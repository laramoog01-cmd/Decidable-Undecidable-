Decidable Problems

A decidable problem has an algorithm that can provide a correct 'yes' or 'no' answer for all inputs and halts after a finite number of steps.

‚úì Algorithm always terminates
‚úì Produces correct answer
‚úì Works for all possible inputs
‚úì Deterministic solution exists

Undecidable Problems

An undecidable problem is one for which no algorithm can be constructed that always provides a correct 'yes' or 'no' answer for every input.

‚úó No universal algorithm exists
‚úó May run forever on some inputs
‚úó Cannot guarantee termination
‚úó Fundamental computational limits

‚àû The Halting Problem

The Problem:

"Given a program and an input, can we determine whether the program will eventually stop or run forever?"

Alan Turing proved that there can be no universal algorithm capable of solving this problem for all programs.

This means that even with unlimited computing power, certain outcomes cannot be predicted algorithmically.

‚ùå Examples of Undecidable Problems

Program Equivalence

Determining if two arbitrary programs will produce the same output for all inputs

Why undecidable: Would require solving the Halting Problem as a substep

Post Correspondence Problem (PCP)

Checking if a sequence of string pairs can be arranged to form matching concatenations

Why undecidable: No general algorithm exists for all cases

‚ùå Additional Undecidable Problems

Rice's Theorem

Any non-trivial property about the behavior of programs is undecidable

Why undecidable: Generalizes many undecidability results

Tile Problem

Can a given set of tiles cover an infinite plane?

Why undecidable: No algorithm can decide this for all tile sets
